## 门面模式

#### 定义

要求一个子系统内部和外部的通信必须通过一个统一的对象进行。门面模式开放一个高层次的接口让外部使用。

#### 实现

```
public class System{
	//开放的外部接口
	public void door(){
		first();
		second();
		third();
	}

	private void first(){}
	private void second(){}
	private void third(){}
}
```

#### 应用

简单说就是构建黑箱，或者模块，封装其中的复杂逻辑。

#### End

子系统内部不能使用门面方法，因为无法修改，所以门面下的子系统需要是确定的系统。
